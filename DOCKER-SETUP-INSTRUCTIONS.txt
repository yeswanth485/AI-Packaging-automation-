================================================================================
DOCKERFILE SPECIFICATIONS
================================================================================

Multi-stage Docker build with:

STAGE 1 - Builder:
  - Base: node:20-alpine
  - Installs: python3, make, g++, openssl (for native dependencies)
  - Copies package files and installs ALL dependencies (--legacy-peer-deps)
  - Generates Prisma client
  - Builds frontend (Next.js)
  - Builds backend (TypeScript → JavaScript)

STAGE 2 - Production:
  - Base: node:20-alpine
  - Creates non-root user (nodejs:1001)
  - Installs ONLY production dependencies
  - Copies built artifacts from builder stage
  - Exposes port 3000
  - Health check on /health endpoint
  - CMD: Runs Prisma migrations then starts Node.js server

RAILWAY DEPLOYMENT STEPS
================================================================================

1. CREATE NEW PROJECT IN RAILWAY
   - Go to https://railway.app/dashboard
   - Click "New Project"
   - Select "Deploy from GitHub repo"
   - Choose: yeswanth485/AI-Packaging-automation-

2. ADD POSTGRESQL DATABASE
   - In your Railway project, click "+ New"
   - Select "Database" → "PostgreSQL"
   - Railway will auto-create DATABASE_URL environment variable

3. ADD REDIS DATABASE
   - In your Railway project, click "+ New"
   - Select "Database" → "Redis"
   - Railway will auto-create REDIS_URL environment variable

4. CONFIGURE ENVIRONMENT VARIABLES
   Go to your service → "Variables" tab and add:

   Required (you must set these):
   ✅ NODE_ENV=production
   ✅ PORT=3000
   ✅ JWT_SECRET=<generate-a-secure-random-string>
   ✅ SESSION_SECRET=42cd5d1cf7c110124e309b8bdf7316ada2a6792d7a266db659a102e7d71ce02b

   Auto-created by Railway (don't touch):
   ✅ DATABASE_URL (from PostgreSQL service)
   ✅ REDIS_URL (from Redis service)

5. CONFIGURE BUILD SETTINGS
   - Go to service → "Settings" tab
   - Builder: Select "Docker" (NOT Nixpacks)
   - Root Directory: Leave empty (Dockerfile is in project root)
   - Dockerfile Path: Dockerfile

6. DEPLOY
   - Railway will automatically trigger a build
   - Monitor the build logs for any errors
   - Wait for deployment to complete (usually 5-10 minutes)

7. VERIFY DEPLOYMENT
   - Click on your service to get the public URL
   - Visit: https://your-app.railway.app/health
   - Should return: {"status":"ok","timestamp":"..."}

================================================================================
TROUBLESHOOTING
================================================================================

If build fails:
1. Check Railway build logs for specific error
2. Verify Dockerfile exists in repository (check GitHub)
3. Verify environment variables are set correctly
4. Check DATABASE_URL and REDIS_URL are auto-created

If deployment succeeds but app crashes:
1. Check Railway deployment logs
2. Verify Prisma migrations ran successfully
3. Check DATABASE_URL connection
4. Verify all environment variables are set

If health check fails:
1. Check if port 3000 is exposed correctly
2. Verify backend server started successfully
3. Check deployment logs for startup errors
